## 製品の特徴（4本柱）

| 特徴 | 説明 |
|------|------|
| ヘッドレス構成 | フロントエンド（Next.js等）とCMSを分離。デザイン・技術選定が自由 |
| APIファースト | REST / GraphQL API + Webhook 標準装備。外部システム連携が容易 |
| マルチドメイン・マルチテナント | 複数サイト・複数組織を1基盤で管理。グループ展開に最適 |
| スケーラビリティ | CDNキャッシュ活用により高負荷・バーストトラフィックに対応 |

---

## 主な標準機能

| 機能カテゴリ | 機能内容 |
|------------|---------|
| コンテンツ管理 | コンテンツ型の自由定義・記事/データ登録・公開ワークフロー・多言語対応 |
| API配信 | REST API / GraphQL API / Webhook（外部システムへのPush通知）|
| 認証・権限 | 会員登録・ログイン・ロール別アクセス制御・SSO対応 |
| メール・通知 | トリガーメール・一斉配信・フォーム送信後の自動通知 |
| ファイル管理 | 画像・動画・PDF等のメディアアップロード・CDN配信 |
| セキュリティ | HTTPS・WAF・定期バックアップ・自動アップデート（SaaS管理）|
| 管理画面 | ノーコードでコンテンツ型設計・直感的な入力UI |

## 導入の流れ（標準）

1. **ヒアリング** ── 現状サイト構成・要件・予算をヒアリング
2. **現状分析・診断** ── ヘッドレスCMS適性評価・技術要件整理
3. **提案・設計** ── コンテンツ設計・フロント技術選定・費用見積
4. **開発・制作** ── Kuroco設定 + フロントエンド実装
5. **テスト・公開** ── 動作確認・パフォーマンステスト後に本番公開
6. **継続サポート** ── 運用支援・機能追加・Kuroco自動アップデート

---

## 提案時の注意事項

- 料金は公開されていない → 「お問い合わせで見積」で案内する
- フロントエンド開発（Next.js等）は別途開発費が発生する旨を説明する
- Kuroco標準機能でできること / カスタム開発が必要なことを明確に区別する
- 既存システム（CRM・MA・予約管理等）とのAPI連携可能性を訴求ポイントにする
