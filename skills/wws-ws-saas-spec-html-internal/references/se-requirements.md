## スキル名定義（フロー固有名称）

| スキル名 | 英名 | 区分 | 概要 |
|---------|------|------|------|
| **乗車地検索フロー** | `BoardingSearchFlow` | F-02/B-01 | 交通種別→会社→路線の3段階マスタ選択＋GPS現在地から停留所を特定するフロー |
| **乗降ペア判定** | `BoardingPairJudge` | B-01 | 乗車地・降車地の2点スタンプが揃って初めて1ptを付与するロジック |
| **乗車中セッション** | `RidingSession` | B-01 | 乗車スタンプ取得〜降車スタンプ取得（またはキャンセル）まで乗車状態を保持する仕組み |

# SE向け機能要件定義 リファレンス
## 対象：WWSプロダクト全般 ／ SmartPRシリーズ ／ 業種・依頼者不問

## ID体系

| プレフィックス | 区分 | バッジ色 |
|-------------|------|---------|
| F-xx | フロントエンド開発 | 🔴 赤（id-cust） |
| B-xx | バックエンド開発 | 🔴 赤（id-cust） |
| B-xx（設定のみ） | 管理画面設定作業 | 🟡 黄（id-ops） |

## 要件テンプレート

### F系（フロント）

```
【F-01】マイページ：ボタン追加＋乗車中状態UI制御
区分：フロント
難易度：小〜中
要件：①乗車/タクシーボタン2種分岐
　　　②乗車中バー（オレンジ・乗車地名表示）
　　　③「降りる」解放・「乗る」グレーアウト制御
　　　④降車完了/キャンセルで状態リセット
　　　⑤キャンセル確認モーダル
確認：マイページHTMLカスタマイズ可能範囲
```

```
【F-02】乗降選択〜停留所検索フロー（全画面）
区分：フロント
難易度：中〜大
要件：①乗る/降りる選択（降りるは乗車後に解放）
　　　②交通種別選択（バス/電車/路面電車/フェリー）
　　　③会社マスタ選択（種別紐づきのDB一覧を表示）
　　　④GPS現在地取得→停留所検索API呼び出し
　　　⑤停留所候補一覧（距離順・最寄りハイライト）
　　　⑥乗車スタンプ確認・取得
　　　⑦降車フロー（会社引継ぎ・GPS再取得）
　　　⑧完了画面・自動リセット
　　　⑨キャンセル確認モーダル
確認：停留所検索API実装方式・会社マスタ管理画面の設定範囲
```

```
【F-03】SSO連携（フロント）
区分：フロント
難易度：中
要件：Card→Receico遷移時、LINEトークンを引き継いで
　　　再ログイン不要にする。
確認：両プロダクトのLINEチャネルが共通か確認（実装方式の分岐点）
```

```
【F-04】審査ステータス表示
区分：フロント
難易度：小〜中
要件：マイページ履歴に「審査中（オレンジ）」「承認済（グリーン）」
　　　バッジを表示。審査中ポイントは合計に含めない。
確認：CardマイページのHTMLカスタマイズ可能範囲を確認
```

```
【F-05】応募モーダル：ポイント内訳表示
区分：フロント
難易度：小
要件：応募モーダル内にポイント内訳（GPS分／タクシー承認済／審査中）を表示。
　　　審査中分はグレーアウトで応募不可。
確認：なし
```

### B系（バックエンド）

```
【B-01】会社マスタ・停留所DB・ペア判定・セッション管理
区分：バック
難易度：大
要件：①交通会社マスタ（種別紐づけ・管理画面CRUD）
　　　②停留所DB（会社ID・名称・緯度経度・判定半径）
　　　③現在地→停留所検索API（lat/lng/company_id/radius→距離順リスト）
　　　④乗降セッション（riding/completed/cancelled状態管理）
　　　⑤降車スタンプは同セッション内の乗車スタンプ取得が前提
　　　⑥ペア成立時1pt付与・片方のみは付与しない
　　　⑦手動キャンセル/降車完了でセッションをリセット
　　　⑧インターバル制限（同一停留所1日1回）
　　　⑨GTFSインポート機能（別途見積）
確認：①セッション管理がCard標準対応か
　　　②GTFSインポートの標準対応有無
　　　③停留所DBのカスタムテーブル追加が可能か
```

```
【B-02】Receico→Card ポイント連携
区分：バック
難易度：中〜大
要件：Receico事務局承認を契機にCardへ1pt付与。
　　　Webhook方式が理想。未対応の場合は日次バッチで代替。
　　　失敗時リトライ3回＋エラーログ。
確認：ReceicoにWebhookが存在するか確認（実装方式の分岐点）
```

```
【B-03】Card→Receico SSO連携
区分：バック
難易度：中
要件：LINEトークン引き継ぎ。チャネル共通の場合はトークン渡し、
　　　別チャネルの場合はワンタイムトークン方式。
確認：両プロダクトのLINEログインチャネルが共通か確認
```

### 設定作業（開発不要）

```
【B-04】OCRタグ審査条件設定
区分：設定作業のみ（コーディング不要）
難易度：設定のみ
要件：管理画面でタクシー会社名リスト・期間条件・重複チェックを設定。
　　　不正チェックバッチを1日1回実行に設定。
確認：対象事業者リストをクライアントから入手
```

## 難易度判定基準

| 難易度 | 目安 | 例 |
|--------|------|-----|
| 大 | 新規ロジック実装が必要 | 乗降ペア判定、外部API連携 |
| 中〜大 | 新規＋既存変更 | Webhook連携、ポイント同期 |
| 中 | 既存機能への追加・連携処理 | SSO、バッチ処理 |
| 小〜中 | 軽微な追加 | 審査ステータス表示 |
| 小 | UI変更・文言変更レベル | ボタンテキスト変更、ラベル追加 |
| 設定のみ | コーディング不要 | 管理画面のタグ条件設定 |
