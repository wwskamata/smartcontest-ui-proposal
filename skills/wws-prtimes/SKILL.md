# WWS PR TIMES プレスリリース生成スキル

PR TIMESは日本最大級のプレスリリース配信サービス。
適切なフォーマットと構成で作成することで、メディア掲載率と読者の行動率が高まる。

## ステップ1：情報収集

着手前に以下を確認・整理する（会話から読み取れるものは確認不要）。

| 必須情報 | 例 |
|---------|-----|
| 製品・サービス名 | MapPenguin Pro |
| 発表の種類 | キャンペーン開始 / 新機能 / 導入事例 / 新規リリース |
| ターゲット業種・規模 | 工場・製造業 / 中規模工場向け |
| 核となるメッセージ（1文） | まずは半年、無料でお試し導入 |
| 期間・数値・価格 | 7月31日まで、¥388,000相当無料 |
| 導入事例（あれば） | AGC / ハルナプロデュース / 大阪メトロ など |
| 担当者・連絡先 | 政森・鎌田 / masamori@ww-system.com |

## ステップ2：PR TIMESフォーマット規則

詳細は `references/prtimes-format.md` を参照。

**絶対に守るルール：**
- タイトル：**40字以内**（全角・半角・英数字・記号・スペースを含めすべて1文字カウント）
- リード文：**200字以内**
- 見出しは `##` 区切りで階層化（PR TIMESのエディタと対応）
- 表は Markdown テーブル形式で記述（PR TIMESはテーブル対応済み）

### タイトルの文字数チェック手順（必須）

タイトル候補を作ったら、**必ず以下の方法で自己検証してから出力すること**。

1. タイトル文字列を1文字ずつ声に出して数える（スペース・記号も1文字）
2. 40字を超えていたら削る。削れない場合は構造を変える
3. 最終的なタイトルに `（XX字 ✅）` または `（XX字 ❌40字超）` を必ず付記する

**よくある超過パターンと回避策：**

| 超過パターン | 回避策 |
|------------|--------|
| 会社名＋製品名＋特典内容を全部入れる | 最も伝えたい1つだけに絞る |
| 「〜キャンペーン開始」で終わらせようとして詰め込む | 動詞を短くする（「実施」→「開始」、「提供します」→削除） |
| 日付と価格と製品名を全部入れる | 日付か価格どちらかを本文に移す |

**40字以内タイトルの良い例：**
- `工場DXをゼロコストで。MapPenguin Pro、7月末まで無料。`（28字）
- `上野動物園がMapPenguinを採用。11言語で園内案内を実現。`（30字）
- `岡山県が公共交通スタンプキャンペーン開始。アプリ不要。`（28字）

### リード文のフォーマット（統一形式）

リード文は必ず以下の区切りで記述すること：

```
## リード文

[本文・200字以内]

（XX字）
```

- 文字数を末尾に括弧で記載する
- 改行・見出しは文字数に含まれない
- 200字を超えた場合は削って200字以内に収めてから出力する

## ステップ3：文書構成（標準テンプレート）

以下の順序で構成する。案件の性質に応じてセクションを追加・削除してよい。

```
## タイトル案（複数案提示 → 最終は1案）
## リード文
## ▼ 背景：[業界の課題]
## ▼ [製品名] とは
## ▼ 主な特長
## ▼ 導入事例
## ▼ [キャンペーン名 or 発表内容] の詳細
## ▼ 料金プラン（あれば）
## ▼ 導入の流れ（あれば）
## ▼ 詳細・お申込み
## ■ 会社概要
## ■ 本件に関するお問い合わせ
## ■ PR TIMES 入稿チェックリスト
```

## ステップ4：各セクションの書き方

### タイトル（最重要）
- 「誰が・何を・どんな効果か」を1行で表現
- 数字・固有名詞を入れるとクリック率が上がる
- 推奨パターン：`[製品名]が[対象]向けに[特典/成果]。[期限/条件]。`
- **3案提示して選んでもらうか、最良案1つを明記して提案する**

### リード文（ポイント）
- 第1文：主語＋発表内容（会社名・製品名・何を発表するか）
- 第2文：なぜ今か・背景
- 第3文：主な特典・数値・読者へのベネフィット
- 200字ピッタリに近づけるのが理想（少なすぎると情報不足）

### 導入事例（説得力の核心）
- 会社名・業種・導入目的・効果を明記
- 可能であれば数値を入れる（「○工場で運用中」「○%削減」等）
- 社名公表NGの場合は「大手○○メーカー」等に変換

### 会社概要（定型）
WWSの定型情報：
- 会社名：株式会社 World Wide System
- サービスサイト：https://maps.smartpr.jp/pro/（MapPenguinの場合）
- ※代表者名・設立日・所在地・資本金はプレースホルダーで出力（`[代表者名]` 等）

## ステップ5：出力と保存

1. 完成したプレスリリースを **Markdown形式** でチャットに表示する
2. 以下のパスに保存する：
   ```
   C:\Users\kamat\ObsidianVault\WWS\提案案件\[案件フォルダ]\prtimes_draft.md
   ```
   案件フォルダが特定できない場合はユーザーに確認する。
3. 最後に「入稿チェックリスト」を必ずつける

## ライティングの原則

- **業界メディアを意識する**：IT系・製造業系・経営系メディアの記者が読んで「取り上げたい」と思う文体
- **読者は2種類**：メディア記者（客観的事実・数値重視）とエンドユーザー（自分ごと化できるか）
- **カタカナ・英字の使い分け**：DX・SaaS・API等の業界用語は使ってよいが、初出は一度説明を添える
- **¥記号と数値**：必ず具体的な数字を使う。「費用削減」より「年間¥240,000削減」
- **受動態を避ける**：「〜が実現されます」より「〜を実現します」
