# PR TIMES 入稿フォーマット詳細リファレンス

## タイトル規則

| ルール | 詳細 |
|--------|------|
| 文字数 | 全角40字以内（半角英数は1文字換算） |
| NG表現 | 「世界初」「業界最高」等の根拠なき最上級表現 |
| 推奨 | 数字・固有名詞・期限を入れるとCTR向上 |
| 例 | 「工場の業務効率化をデジタルマップで実現。7月31日まで初期費用無料キャンペーン開始」 |

## リード文規則

| ルール | 詳細 |
|--------|------|
| 文字数 | 200字以内 |
| 構成 | ①主語+発表内容　②背景・なぜ今か　③主な特典・数値 |
| 目的 | 検索エンジン・ニュースフィードでの要約表示に使われる |
| NG | 「〜について」で始める / 社内向け表現（「弊社は〜」ではなく「株式会社○○は〜」） |

## 本文構成パターン別

### キャンペーン開始の場合
1. 背景（業界課題）
2. サービス概要
3. キャンペーン詳細（特典一覧・期間・対象）
4. 導入事例
5. 料金プラン
6. 導入フロー
7. 申込先URL

### 新製品・新機能リリースの場合
1. リリースの概要（何を・なぜ・いつから）
2. 解決する課題
3. 機能詳細
4. 競合との差別化ポイント
5. 対応プラン・価格
6. ベータユーザー募集（あれば）

### 導入事例発表の場合
1. 導入企業プロフィール
2. 導入前の課題
3. 選定理由
4. 導入内容・活用方法
5. 導入後の効果（数値）
6. 担当者コメント（あれば）

## カテゴリ設定（PR TIMESのカテゴリ選択）

WWSプロダクトに推奨するカテゴリ：
- **メイン**：IT・通信・インターネット
- **サブ**：製造・ものづくり（工場向け）/ 流通・小売（スタンプ系）/ 観光・旅行（施設向け）

## 画像添付ガイドライン

PR TIMESへの画像は必須（ない場合、掲載率・CTRが大幅低下）。

| 用途 | 推奨サイズ | 内容 |
|------|-----------|------|
| アイキャッチ（必須） | 1200×630px | 製品ロゴ＋キャッチコピー |
| サービス画面 | 800×600px以上 | 実際の操作画面スクリーンショット |
| 導入事例画像 | 800×600px以上 | 導入先ロゴ・施設写真（許諾取得済みのもの） |
| インフォグラフィック | 任意 | 料金比較・導入フロー図 |

## 配信設定チェックリスト

```
□ タイトル：40字以内（文字数カウンターで確認）
□ リード文：200字以内
□ 本文：見出しの階層が正しい（H2→H3の順）
□ 画像：最低1枚、1200×630px以上のアイキャッチ添付
□ 会社概要：代表者名・所在地・設立年月日・資本金を記入
□ カテゴリ：メイン＋サブカテゴリを設定
□ キーワード：3〜5つ設定
□ 配信日時：即時 or 指定日時を選択
□ 配信先メディア：対象業種を選択
□ 校正：社名・製品名・数字に誤りがないか最終確認
□ URL：リンク先ページが正常動作するか確認
□ プレビュー：スマホ表示でも読みやすいか確認
```
