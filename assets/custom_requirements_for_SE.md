## スキル名定義

| スキル名 | 英名 | 概要 |
|---------|------|------|
| **乗車地検索フロー** | `BoardingSearchFlow` | 交通種別→会社→路線の3段階マスタ選択 ＋ GPS現在地検索で停留所を特定するフロー |

### 乗車地検索フローの画面構成（5STEP）
| STEP | 画面 | 内容 |
|------|------|------|
| STEP 1 | 乗る/降りる選択 | 降りるは乗車後に解放 |
| STEP 2 | 交通種別選択 | バス・電車・路面電車・フェリー |
| STEP 3 | 会社選択 | 種別に紐づくマスタ一覧 |
| STEP 3-b | **路線選択** | 会社に紐づく路線マスタ（路線番号・区間） |
| STEP 4 | GPS現在地取得 | 路線で絞り込んだ停留所を検索 |
| STEP 5 | 停留所候補一覧 | 距離順・最寄りハイライト |

### 【F-01】マイページ：乗車スタンプボタン ＋ 乗車中状態の表示制御
**画面：** SmartStampCard マイページ TOP
**区分：** フロント

#### 要件
**① ボタンの2種分岐**
- 標準の汎用スタンプボタンを以下2種に変更
  - `[🚃 乗車スタンプを押す]` → 乗降選択フロー（F-02）へ遷移
  - `[🧾 タクシーレシートを登録する]` → SmartRecieco SSO遷移（F-03/B-03）

**② 乗車中状態のUI制御**（乗車スタンプ取得後〜降車スタンプ取得まで）
- **乗車中バー**（オレンジ）をマイページ上部に固定表示
  - 表示内容：「🚌 乗車中：{乗車地名}」＋「キャンセル」リンク
- `[降りる（降車スタンプを押す）]` ボタンを有効化（オレンジ）
- `[乗る]` ボタンをグレーアウト・タップ不可（乗車中ロック）
- 降車スタンプ取得完了または手動キャンセルで状態をリセット

#### 状態遷移
```
通常状態：[乗車スタンプを押す]（赤）　[タクシーレシート]（アウトライン）
　↓ 乗車スタンプ取得
乗車中状態：乗車中バー（オレンジ）
　　　　　　[降りる]（オレンジ・有効）
　　　　　　[乗る]（グレー・無効）
　↓ 降車スタンプ取得 or キャンセル
通常状態に戻る（リセット）
```

#### 備考
- 乗車中状態はブラウザセッション内で保持（B-01と連動）
- キャンセル時は確認モーダル（⑫画面）を表示して誤操作防止

### 【F-02】GPS スタンプ：**乗車地検索フロー**（乗降選択〜停留所検索フロー 全画面）
**画面：** SmartStampCard スタンプ取得フロー（全12画面）
**区分：** フロント

#### フロー概要
```
① 乗る/降りる選択
  → ② 交通種別選択（バス/電車/路面電車/フェリー）
    → ③ 交通会社選択（マスタから選択）
      → ③-b 路線選択（会社に紐づく路線マスタから選択）
        → ④ GPS現在地取得
        → ⑤ 停留所候補一覧（距離順）
          → ⑥ 乗車スタンプ確認・取得
            → 乗車中状態（マイページに戻る）
              → ⑦ 降車：GPS再取得→停留所選択
                → ⑧ 降車スタンプ確認・取得
                  → ⑨ 完了（ペア成立・1pt・自動リセット）
```

#### 各画面の要件

**② 乗る/降りる選択画面**
- 「乗る」「降りる」を2択カードで表示
- 「降りる」は乗車スタンプ取得前はグレーアウト・タップ不可
- 乗車スタンプ取得後は「降りる」のみ操作可能

**③ 交通種別選択画面**
- バス／電車／路面電車／フェリー の4種をタイルで表示
- 選択後「次へ：会社を選ぶ」ボタンで次ステップへ
- STEPインジケーター（全4STEP）を表示

**④ 交通会社選択画面**（マスタ選択）
- 選択した交通種別に紐づく事業者一覧をDBから取得・表示
  - 例：バス → 両備バス・宇野バス・中国バス・備北バス 等
- 会社名・路線エリアを表示
- 選択した会社はチェックマーク表示
- 「← 種別に戻る」で③に戻れる

**⑤ GPS現在地取得 → 停留所候補一覧**
- GPS有効化ボタン＋設定方法リンクを表示
- 取得成功後、選択した会社の停留所を**距離順**でリスト表示
  - 最寄り停留所をハイライト（緑枠・最寄りバッジ）
  - 各停留所に現在地からの距離を表示（単位：m）
  - 「現在地を再取得する」ボタンを設置
- 検索範囲：半径500m以内（設定値・要確認）

**⑥ 乗車スタンプ確認画面**
- 確認カード：停留所名・会社名・距離・「1point付与（ペア成立時）」
- 乗降の流れ表示（乗車地：選択中 → 降車地：未選択）
- 「応募する」ボタンでスタンプ取得

**⑦ 降車フロー**
- 乗車時に選択した会社を引き継ぎ（再選択不要・UI上で明示）
- GPS再取得 → 停留所候補一覧（乗車時と同様）
- 乗車地・降車地のルート確認表示（乗車地✅ → 降車地←今ここ）

**⑧ 降車スタンプ確認画面**
- 確認カード：降車地名・「ペア成立 → 1pt付与！」を明示
- 乗車地（取得済✅）→ 降車地（今ここ）のルート表示

**⑨ 完了画面**
- 獲得ポイントと乗車区間を表示
- 「乗車中状態をリセットしました」メッセージ
- 「マイページに戻る」ボタン

**⑫ キャンセル確認画面**
- 乗車スタンプが無効になる旨の警告を表示
- 「キャンセルする」「戻る（乗車を続ける）」の2択

#### 備考
- 会社マスタは管理画面で登録・交通種別ごとに紐づけ設定
- 停留所データはGTFSまたは手動登録（B-01参照）

### 【B-01】GPS スタンプ：**乗車地検索フロー** バックエンド（会社マスタ・路線マスタ・停留所DB・ペア判定）
**区分：** バックエンド

#### 要件

**① 交通会社マスタ**
- 交通種別（バス/電車/路面電車/フェリー）に紐づく事業者を管理
- 管理画面から登録・編集可能
```
TransitCompany: {
  id, name, type: "bus"|"train"|"tram"|"ferry",
  area: string,  // 例："岡山市内"
  active: boolean
}
```

**①-b 路線マスタ**
- 会社IDに紐づく路線を管理
- 路線番号・路線名・区間（起点〜終点）を保持
- 管理画面から登録・編集可能（GTFSインポートにも対応）
```
TransitRoute: {
  id, company_id,
  route_no: string,      // 例："01"
  name: string,          // 例："岡山駅〜東山線"
  origin: string,        // 起点停留所名
  destination: string,   // 終点停留所名
  active: boolean
}
```

**② 停留所DB**
- 会社IDに紐づく停留所を管理（GTFSインポート or 手動登録）
- 緯度・経度・半径（GPS判定範囲）を保持
```
Stop: {
  id, company_id, name,
  lat, lng, radius_m,   // GPS判定範囲（デフォルト100m）
  type: "bus_stop"|"station"|"tram_stop"|"ferry_port"
}
```

**③ 現在地からの停留所検索API**
- リクエスト：`{ lat, lng, company_id, radius: 500 }`
- レスポンス：半径内の停留所リストを距離順で返す
```
GET /api/stops/nearby?lat=xx&lng=xx&company_id=xx&radius=500
→ [ { stop_id, name, distance_m, lat, lng }, ... ]
```

**④ 乗降セッション管理**
- セッション単位で「乗車地取得済み」状態を保持
- 降車スタンプは同セッション内に乗車スタンプ取得済みの場合のみ受付
- ペア成立時に1ポイント付与
- 手動キャンセルまたは降車完了でセッションをリセット

```
StampSession: {
  id, user_id,
  company_id,           // 選択した会社
  transit_type,         // 交通種別
  departure_stop_id,    // 乗車地
  departure_at,         // 乗車時刻
  arrival_stop_id,      // 降車地（取得後に更新）
  arrival_at,           // 降車時刻
  status: "riding"|"completed"|"cancelled",
  point_granted: boolean
}
```

**⑤ インターバル制限**
- 同一ユーザー・同一停留所：1日1回まで取得可
- 同一ユーザー：同時に複数のアクティブセッションを持てない

#### 備考
- 停留所検索APIはフロントのGPS取得後に叩く（F-02連動）
- GTFSデータからの一括インポート機能は別途見積もり（B-04参照）

### 【B-02】SmartRecieco → SmartStampCard ポイント連携
**区分：** バックエンド

#### 要件
- SmartRecieco で**事務局が承認操作を行った際**に、SmartStampCard 側に1ポイントを付与する
- 連携方式：**Webhook または API コール**（Receico承認イベント → Card ポイント加算エンドポイント）
- 付与単位：承認1件＝1ポイント（固定）
- 付与失敗時はリトライ処理（最大3回）＋エラーログ記録

#### 連携フロー
```
[事務局 承認操作（Receico管理画面）]
  ↓ Webhook / API
[Card ポイント加算処理]
  ↓
[ユーザーマイページ反映]
```

#### 備考
- Receico側の承認Webhookが存在するか事前確認要
- 存在しない場合は日次バッチ（承認済み差分を取得→Card APIへ送信）で代替

### 【F-03 / B-03】SmartRecieco → SmartStampCard SSO 連携
**区分：** フロント ＋ バックエンド

#### 要件
- CardマイページのタクシーボタンからRecieco画面へ遷移する際、**LINEトークンを引き継いで再ログイン不要**にする
- 遷移後のRecieco画面はタクシー領収書登録専用ページとして表示

#### 実装方式（案）
- Card側でLINEアクセストークンを取得 → URLパラメータ or Cookieで渡す
- Receico側でトークン検証 → セッション生成

#### 備考
- 両プロダクトが同一LINEログインチャネルを共有しているか確認要
- チャネルが別の場合はトークン共有不可 → ワンタイムトークン方式を検討

### 【F-04】マイページ：タクシーレシート審査ステータス表示
**画面：** SmartStampCard マイページ（ポイント履歴エリア）
**区分：** フロント

#### 要件
- ポイント取得履歴に **「審査中」「承認済」** のステータスバッジを表示
  - 審査中：オレンジバッジ＋「2〜3営業日以内に反映」テキスト
  - 承認済：グリーンバッジ＋付与ポイント数
- 審査中ポイントは合計ポイントに**含めない**（確定分のみ表示）

#### データ取得
- Receico側の審査ステータスをCard側に定期同期 or API参照

---

### 【F-05】応募モーダル：ポイント内訳表示
**画面：** SmartStampCard 応募確認モーダル
**区分：** フロント

#### 要件
- 応募モーダル内に確定ポイントの内訳を表示
  - 🚃 乗車スタンプ（GPS）：N pt
  - 🧾 タクシーレシート（承認済）：N pt
  - ⏳ タクシーレシート（審査中）：+N pt 予定（グレーアウト）

#### 備考
- 審査中ポイントはUI表示のみ、応募には使用不可

---

### 【B-04】タクシーレシート：OCRタグ審査条件設定
**区分：** バックエンド（Receico管理画面設定）

#### 要件
- SmartReceico のタグ・審査条件設定で以下を構成（**開発不要・管理画面設定で対応**）
  - タクシー会社名リストを登録（対象事業者のみ審査OK）
  - 乗車日時がキャンペーン期間内であることをチェック
  - 重複チェック：同一応募者の同一レシート（日時＋金額一致）を自動NG
- 全応募に対する不正チェック（バッチ）を1日1回実行に設定

#### 備考
- **コーディング不要**のため工数は設定作業費として計上
- 対象タクシー事業者リストはクライアント（行政）から提供を受ける
