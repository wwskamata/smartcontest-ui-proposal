# Claude.ai プロジェクト セットアップ手順

## 作成するもの

```
Claude.ai のプロジェクト「WWSプロデューサーAI」
├── カスタム指示  ← custom_instructions.md の内容をペースト
└── ナレッジ
    ├── knowledge_skills.md   ← スキル詳細手順
    └── knowledge_products.md ← WWS製品知識
```

---

## STEP 1｜プロジェクトを作成する

1. https://claude.ai にアクセスしてログイン
2. 左サイドバーの「**プロジェクト**」→「**プロジェクトを作成**」
3. 名前を入力：`WWSプロデューサーAI`
4. 「**作成**」をクリック

---

## STEP 2｜カスタム指示を設定する

1. プロジェクト画面右上の「**…（設定）**」または「**プロジェクトを設定**」をクリック
2. 「**カスタム指示**」欄に `custom_instructions.md` の内容をコピー＆ペースト
3. 「**保存**」をクリック

> ポイント：カスタム指示はプロジェクト内の全会話に適用されます。

---

## STEP 3｜ナレッジをアップロードする

1. プロジェクト設定画面の「**コンテンツを追加**」または「**ナレッジ**」セクションへ
2. 以下のファイルをアップロード：
   - `knowledge_skills.md`（スキル手順）
   - `knowledge_products.md`（製品知識）
3. アップロード後に「保存」

---

## STEP 4｜動作確認

プロジェクト内で以下を試す：

```
「SmartStampCardとSmartReciecoを組み合わせたレシートキャンペーンの
 モックアップを作りたいです。クライアントは食品メーカーです。」
```

→ 🟡 コンテキストセットが自動発動し、目的・成果物・制約を整理してから
　 wws-ws-saas-spec-html-internal スキルの手順に沿って提案が始まるはずです。

---

## STEP 5｜チームメンバーと共有する（Teamプランの場合）

1. プロジェクト設定 → 「**メンバーを追加**」
2. 共有したいメンバーのメールアドレスを入力
3. 権限を設定（「**編集者**」または「**閲覧者**」）
4. 招待を送信

> チームメンバーは同じカスタム指示・ナレッジが適用された状態で使えます。

---

## ナレッジの更新方法

### 製品知識が変わったとき
1. `knowledge_products.md` を編集
2. プロジェクト設定のナレッジから該当ファイルを削除 → 再アップロード

### 新スキルを追加したとき
1. `knowledge_skills.md` に新スキルのセクションを追記
2. `custom_instructions.md` のスキル一覧テーブルに新スキルの行を追加
3. 両ファイルをプロジェクトで更新

### 進行中案件情報を追加したい場合
- プロジェクト設定 → 「コンテンツを追加」
- `project_summary_{案件名}.md` や `mockup_planb_transit_campaign.html` をアップロード
- 会話内で「この案件の〜」と話しかければ参照してくれます

## Claude Code と Claude.ai プロジェクトの使い分け

| 用途 | Claude Code | Claude.ai プロジェクト |
|------|------------|---------------------|
| HTMLモックアップ生成 | ✅ ファイル操作が必要 | △ テキスト出力のみ |
| PPTX生成（node実行） | ✅ コード実行が必要 | ✕ 実行不可 |
| GitHub Pages 公開 | ✅ CLI操作が必要 | ✕ 実行不可 |
| 提案内容の相談・レビュー | ◯ | ✅ チーム共有しやすい |
| プレスリリース文章作成 | ◯ | ✅ どこでも使える |
| プロダクト選定・比較 | ◯ | ✅ どこでも使える |
| 社内メンバーへの展開 | ✕ 個人環境依存 | ✅ チーム共有可能 |
